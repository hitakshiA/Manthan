## Design Context

### Users
Data analysts, business users, and hackathon judges evaluating an autonomous data analysis platform. They upload datasets (CSV, Parquet, Excel), ask natural-language questions, and receive structured analytical output — KPI cards, dashboards, and multi-page reports. The context is professional analysis work: they need to trust the output, understand how it was derived, and drill deeper. Judges specifically need to see the full stack working in a 5-minute demo — upload, clarification, live agent activity, and polished output.

### Brand Personality
**Precise, warm, capable.** Manthan is a senior analyst who's thorough but approachable — shows their work without being cold. Think of someone who hands you a well-formatted report with a sticky note saying "page 3 has the surprise."

### Emotional Goals
- **Confidence**: Every number is real, every source is cited, the work is visible
- **Impressed**: The speed and depth surprise — a 6-page report with recommendations in 90 seconds
- **Calm control**: The system asks before guessing, shows its plan, waits for approval
- **Curiosity**: Every answer invites a follow-up via drill-down hints

### Aesthetic Direction
**Warm intelligence** — Linear's density meets Hex's notebook feel. Light mode, warm-tinted neutrals (hue 65 golden undertone), deep indigo as the single accent color. Professional workspace, not a chatbot. No landing page — boot straight into work.

**Primary reference**: Hex / Observable — notebook-style data analysis where code, prose, and charts compose together. The agent activity feed is Manthan's equivalent of a notebook's execution cells.

**Anti-references**: ChatGPT (chat bubbles), generic dashboards (card grids), dark-mode-with-neon (AI slop), glassmorphism, gradient text.

### Design Principles
1. **Show the work, not the worker.** Agent activity is choreography — compact inline tokens that expand on hover, not verbose log cards. The UI communicates what's happening without demanding attention.
2. **Results materialize, not appear.** KPIs count up, charts draw axes first then data flows in, sections stagger with quart-out easing. The transition from "working" to "done" is a layout morph, not a page swap.
3. **Every number has a source.** Citations are collapsible but always present. Provenance badges mark recalled memory. The semantic layer moment (clarification questions) is a designed interaction, not a form.
4. **Density through typography, not cards.** Spacing and font weight create grouping. Cards are used only for truly distinct, actionable items (KPIs, recommendations). Never nest cards inside cards.
5. **One accent, used sparingly.** Deep indigo (oklch 55% 0.18 265) is the single brand color. It marks active states, interactive elements, and the dataset badge. Everything else is warm neutrals. Accent overuse kills its power.

### Existing Design System
- **Font**: Plus Jakarta Sans (400/500/600/700) via Google Fonts
- **Colors**: OKLCH palette with warm-tinted neutrals (hue 65), deep indigo accent (hue 265), semantic green/amber/red
- **Spacing**: 4pt base system (4, 8, 12, 16, 20, 24, 32, 40, 48, 64)
- **Radii**: 6/8/12px (sm/md/lg)
- **Easing**: quart-out (0.25, 1, 0.5, 1), expo-out (0.16, 1, 0.3, 1)
- **Icons**: Lucide React
- **Charts**: ECharts 6 via echarts-for-react
- **State**: Zustand
- **Stack**: Vite + React 19 + TypeScript + Tailwind CSS 4

### Theme
Light mode only. Dark mode is explicitly not planned — the warm neutral palette is the identity.
