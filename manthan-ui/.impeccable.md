## Design Context

### Product
**Manthan** - a hosted AI teammate for the **billing queue** at B2B SaaS companies ($5–50M ARR). Manthan investigates every chargeback, failed payment, refund request, and dunning escalation; joins evidence across payments + CRM + support + product analytics + policy docs; drafts the response; fires the action under approval gates; cites every claim. Standard monthly subscription, no per-event pricing surprises.

### Users
**Heads of RevOps and Finance** at $5–50M ARR B2B SaaS - and their senior operators (CFO, Controller, Head of Customer Operations). They run a billing queue that today is worked by humans copy-pasting between systems at 11 PM on Fridays. They have procurement, security, and audit constraints. They are not "AI-curious" - they need the auditability of a senior analyst, not the vibe of a chatbot.

They pivot fast on three questions:
1. Will my CFO sign a brief Manthan wrote? (must cite everything)
2. What happens when Manthan is wrong? (must gate at thresholds)
3. Is the data handling clean enough for procurement? (encryption, SOC 2, audit log)

### Brand Personality
**Senior analyst, not chatbot.** Manthan is the trusted ops lead who hands you a printed brief with a sticky note: *"page 3 has the surprise."* It shows its work without performing intelligence. It is *quiet, exact, and accountable* - not chatty, not bubbly, not "✨".

The interface should feel like a CFO's daily desk: a single high-signal page with the receipt of every claim already attached. Editorial restraint over console flash.

### Emotional Goals
- **Confidence** - every number is sourced, every action timestamped, every model call audited
- **Calm authority** - the system asks before acting on anything that could blow up
- **Recognition** - the buyer thinks *"this is how I'd build it if I had two more engineers"*
- **Procurement-ready** - the security review takes a week, not a quarter

### Aesthetic Direction
**Operations Memo.** Editorial typographic restraint meets ledger density. Think the *Financial Times* + Linear's spatial discipline + a CFO's bound monthly briefing - *not* a SaaS marketing site.

**Primary references:**
- The Financial Times print/digital edition (rules, footnotes, restrained accent)
- Bloomberg Terminal (information density, tabular numbers, mono for IDs)
- Linear (motion, spatial rhythm) - for engineering polish only, not aesthetic

**Anti-references** (these are what we *avoid*):
- Dark-mode-with-neon-emerald (AI slop)
- Glassmorphism / liquid glass chips (AI slop)
- Sparkles icon on AI avatars (AI cliché)
- Templated sub-tab strips repeated across distinct surfaces (AI cliché)
- Big metric card grid with delta pills (AI cliché - "hero metric layout template")
- Cards nested inside cards
- Glowing accent shadows (`box-shadow: 0 0 X rgba(accent, *)`)
- Centered hero copy
- Inter as the body font
- ChatGPT-style chat bubble demos

### Design Principles

1. **Rules, not boxes.** Major sections separate with a 1px horizontal rule, not a rounded card border. We use cards only when the unit is genuinely standalone and actionable (an approval row, a drafted action).

2. **Numbers are typography.** Currency, case IDs, and amounts use `tabular-nums` mono. Percentages, timestamps, and counts use mono. Prose stays in sans. The shift between sans and mono encodes meaning.

3. **One accent, no glow.** Desaturated emerald `oklch(0.62 0.15 150)` marks *actionable* - primary CTAs, "approved" status, "drafted" badges. Never glow halos. Never as a decoration. Ledger amber `oklch(0.78 0.13 75)` is the only secondary accent, reserved for "awaiting your nod."

4. **Citations are first-class.** Every Manthan claim has a numbered footnote `[1]` `[2]` `[3]` linking to the source record. Footnotes sit at the bottom of the memo, not in tooltips. We never hide provenance behind a hover.

5. **Asymmetric layouts.** Hero is left-aligned. Two-column sections lean 60/40 or 40/60, never 50/50. Whitespace is part of the composition.

6. **Distinct surfaces, distinct shapes.** The case workspace, the approval queue, and the policy editor each have *their own* layout grammar. They are not three tabs of the same template.

7. **Motion is purposeful.** State changes get motion (a row resolving, a brief filling in, a draft animation). Decoration doesn't. No bounce, no elastic - quart-out easing only.

```
--bg            oklch(0.135 0.006 75)   /* warm dark, not pure black */
--surface       oklch(0.165 0.006 75)   /* elevated panel */
--surface-2     oklch(0.195 0.006 75)   /* row hover, deeper card */
--rule          oklch(0.30 0.005 75)    /* horizontal rule, divider */
--rule-soft     oklch(0.22 0.005 75)    /* subtle divider */
--ink           oklch(0.95 0.004 75)    /* warm white, not pure */
--ink-strong    oklch(0.98 0.003 75)    /* headlines */
--ink-muted     oklch(0.68 0.006 75)    /* body muted, captions */
--ink-faint     oklch(0.50 0.006 75)    /* labels, footnotes */

--accent        oklch(0.62 0.15 150)    /* desaturated emerald - actions */
--accent-soft   oklch(0.62 0.15 150 / 0.12)
--accent-ink    oklch(0.18 0.05 150)    /* text on accent surface */

--amber         oklch(0.78 0.13 75)     /* "awaiting your nod" - only here */
--amber-soft    oklch(0.78 0.13 75 / 0.12)

--danger        oklch(0.62 0.20 28)
--info          oklch(0.70 0.10 240)
```

### Type System
- **Display (italic accents)**: Instrument Serif - used for emphasis words inside headlines and section labels
- **Body**: **Geist Sans** - 400 / 500 / 600 - Vercel's typeface, distinctive without being trendy
- **Mono**: **Geist Mono** - for amounts, case IDs, timestamps, code
- **Never**: Inter, Roboto, Arial, system-ui defaults

### Spacing
4pt base. Use `clamp()` for fluid section padding. Vary spacing between sections - generous breathing room between major bands, tight within rows.

### Motion
- Quart-out: `cubic-bezier(0.25, 1, 0.5, 1)`
- Expo-out: `cubic-bezier(0.16, 1, 0.3, 1)`
- Durations: 180–600ms. No bounce. No elastic.

### Theme
Dark only. The warm-tinted dark is the brand. Light mode is not on the roadmap.

### Stack
- Vite + React 19 + TypeScript + Tailwind CSS 4
- `motion/react` for animations
- `lucide-react` for icons (selectively - no Sparkles, no decoration icons)
- `simple-icons` for SaaS brand logos
- Zustand for state (when state is needed)
