# Data Context Document (DCD) Schema

The DCD is Manthan's semantic contract with downstream analysis agents.
It is a pydantic-validated YAML document produced by
`src.semantic.generator.build_dcd()` and consumed by every agent tool.

This page documents the wire format; the canonical schema lives in
`src/semantic/schema.py`.

## Envelope

```yaml
version: "1.0"
dataset:
  id: "ds_abcdef1234"
  name: "Nexaretail Sales"
  description: "CSV dataset loaded from nexaretail_sales.csv (500 rows, 12 columns)."
  source: { ... }
  temporal: { ... }
  columns: [ ... ]
  computed_metrics: [ ... ]
  relationships: [ ... ]
  quality: { ... }
  verified_queries: [ ... ]
  agent_instructions: [ ... ]
```

## dataset.source

```yaml
source:
  type: "csv"                        # csv | excel | json | parquet | postgres | mysql | sqlite
  original_filename: "sales.csv"
  ingested_at: "2026-04-11T14:30:00Z"
  row_count: 1500000
  raw_size_bytes: 245000000
```

## dataset.temporal

```yaml
temporal:
  grain: "daily"                     # daily | weekly | monthly | quarterly | yearly | irregular | null
  column: "order_date"               # Name of the temporal column (null if none)
  range:
    start: "2024-01-01"
    end: "2026-03-31"
  timezone: "UTC"
```

## dataset.columns[]

```yaml
- name: "revenue"
  dtype: "DOUBLE"                    # DuckDB type string
  role: "metric"                     # metric | dimension | temporal | identifier | auxiliary
  description: "Total order amount in USD"
  aggregation: "SUM"                 # SUM | AVG | COUNT | MIN | MAX | null (metrics only)
  nullable: false
  completeness: 1.0                  # 0.0 - 1.0
  cardinality: 14320
  sensitivity: "public"              # public | quasi_identifier | pii
  pii_type: null                     # e.g. EMAIL_ADDRESS, PERSON, PHONE_NUMBER
  handling: null                     # expose | mask_in_outputs | aggregate_only | never_expose_in_outputs
  stats:
    min: 0.50
    max: 24999.99
    mean: 142.37
    median: 89.50
    stddev: 203.14
    p25: 45.00
    p75: 178.00
  sample_values: [125.50, 89.00, 299.99, ...]
```

## dataset.computed_metrics[]

```yaml
- name: "average_revenue_per_order_id"
  formula: "SUM(revenue) / COUNT(DISTINCT order_id)"
  description: "Average revenue per unique order"
  depends_on: ["revenue", "order_id"]
```

## dataset.quality

```yaml
quality:
  overall_score: 0.92                # 0.0 - 1.0
  freshness:
    last_record_date: "2026-03-31"
    expected_frequency: "daily"
    status: "fresh"                  # fresh | stale | unknown
  completeness:
    fully_complete_columns: 14
    partial_columns: 2
    details:
      - column: "customer_name"
        completeness: 0.87
        note: "13% missing"
  known_limitations:
    - "Revenue excludes returns processed after 30 days"
  validation_rules:
    - "revenue >= 0"
```

## dataset.verified_queries[]

Auto-generated by `src.materialization.query_generator`. Each entry
targets the Gold table directly and is guaranteed to execute against
the current schema.

```yaml
- question: "What is the total revenue by region?"
  sql: "SELECT \"region\" AS region, SUM(\"revenue\") AS total_revenue FROM gold_sales_abcdef1234 GROUP BY \"region\" ORDER BY total_revenue DESC"
  intent: "breakdown"                # breakdown | summary | trend | change | comparison
```

## dataset.agent_instructions[]

Natural-language directives injected into downstream analysis agents'
system prompts. Examples emitted by the generator:

- "Never include the following PII columns in query outputs: customer_email, customer_name"
- "Always aggregate 'revenue' using SUM."
- "Surface data-quality warnings when querying columns with completeness below 95%: customer_name"

## Validation

Every DCD is validated by pydantic on construction (`DataContextDocument(
**data)`) and optionally against the DuckDB catalog via
`src.semantic.editor._verify_columns_against_catalog` when applied as an
edit. Unknown column references and role-enum violations fail with
`DcdValidationError`.
